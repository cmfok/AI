## 一、总体指南

### 1.1 工作方式
- 基于「完整框架.md」（演讲稿）重新生成 PPT 大纲
- 参考「PPT大纲.md」（当前大纲）的结构和内容
- **逐页讨论确认**：每页我先给建议，用户确认后记录
- 最终输出：`分享会ppt大纲.md`

### 1.2 注意事项

| # | 说明 |
|:--:|------|
| 1 | ❌ 不定义每页视觉风格，由最终使用的 Skill 接管 |
| 2 | 💬 每页我先建议元素和布局类型，用户决定，我记录 |
| 3 | 📌 每页的页面编号、核心标题必须双方确认 |
| 4 | 💡 我可以从听众观感角度提建议，但用户确认的事按用户要求记 |
| 5 | 📝 所有记录必须详细，不能缩减 |
| 6 | 📖 如需改演讲稿，先提「从什么改成什么」的建议 |
| 7 | 🔍 案例的详细描述、核心信息不能缩减 |

### 1.3 设计系统参考（不在大纲中定义风格）

**可选 Skill：**
- `html-ppt-full`：34+ 主题、31 种布局、27 种 CSS 动画 + 20 种 Canvas FX
- `guizang-ppt-skill`（瑞士国际主义风格）：22 个登记版式(S01-S22)、4 套主题色、22 种动效
- `ppt-master`：SVG→PPTX 工作流

**已有实现参考：**
- `分享会/html ppt/index.html` — aurora 主题的 24 页 HTML 幻灯片
- `分享会/web/` — 自定义 Web 展示框架（含控制台、Q&A、语音识别）

### 4.1 现有架构
```
控制台（tech_control.html）             展示页（tech_display.html）
  └─ POST /api/control/go                 └─ 轮询 GET /api/control/current
       {slide: N}                               → 渲染对应 Slide
  └─ POST /api/control/qa-show            └─ 轮询 GET /api/control/qa-overlay
  └─ POST /api/qa/ask                     └─ 轮询 GET /api/questions
  └─ GET /api/questions
```

### 6.7 四大工程生态栏递进关系（分层堆叠）

每个工程在上一个工程的生态栏基础上叠加新东西，最终呈现为**分层堆叠**，不是一行平铺：

```
循环工程的生态栏（最高层，最完整）：

  ⏰ Cron · 🤖 Agent · 📊 日志        ← 第4层：循环工程新增
  🐍 Python · 🔗 Coze · 🔧 Dify     ← 第3层：规训工程新增
  📄 MD文件 · 📓 Obsidian · 📘 飞书   ← 第2层：上下文/知识库
  🧠 ChatGPT · Kimi · 豆包 · ...    ← 第1层：提示词工程
  ─────────────────────────────────
  越往下越基础，越往上越进阶
```

每个工程页的生态栏只显示到对应层：
- 提示词工程页 → 只显示第1层
- 上下文工程页 → 显示第1~2层
- 规训工程页   → 显示第1~3层
- 循环工程页   → 显示第1~4层（完整堆叠）
