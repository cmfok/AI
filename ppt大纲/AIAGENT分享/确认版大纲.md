# 强制执行要求

> ⚠️ 以下要求为硬性规定，开发和制作过程中必须严格遵循，不得偏离。

## 一、适配与交互

| # | 要求 | 说明 |
|:--:|------|------|
| **F1** | 适配手机版本 | 展示页必须响应式适配移动端，按钮够大、间距充足，适合触控操作 |
| **F2** | 手机滑动翻页 | 移动端必须支持左右滑动切换页面（touch swipe） |
| **F3** | 键盘按键翻页 | 桌面端必须支持键盘 ← → 方向键翻页（上一页/下一页） |
| **F4** | 键盘快捷键 | `1/2/3` Slide 4 展开对应三阶段卡片；`4` 展示局限案例；`0` 收起；`E` Slide 2 切换答案卡片 |

## 二、页码与结构（硬性规定）

| # | 要求 | 说明 |
|:--:|------|------|
| **P1** | 总页数 21（不含附录） | Slide 1~21 为正页，附录显示为"附" |
| **P2** | 第 5 页 = 转折·四大工程总览 | 不是模型对比表！模型对比表移至附录 A |
| **P3** | 第 19 页 = 落地 Checklist + 配套工具包 | 保留工具包三层内容，不删除 |
| **P4** | 附录结构 | 附录A：模型对比表 / 附录B：Token费用（占位页，先不做） |

## 三、控制台规范

| # | 要求 | 说明 |
|:--:|------|------|
| **C1** | 固定置顶 | 页码栏 + 页码网格 `position: sticky; top:0`，不随滚动移动 |
| **C2** | 固定底部 | 翻页按钮 `position: fixed; bottom` |
| **C3** | 仅演讲稿可滚动 | 其他区域全部固定 |
| **C4** | 手机使用 | 演讲者手机打开控制台控制 PPT 翻页 |
| **C5** | 页码网格 | 显示全部页码 P1~P20，点击任意页码跳转 |
| **C6** | 演讲稿面板 | 控制台下方显示当前页 + 下一页演讲稿（上下排列，方便预读） |
| **C7** | 操作按钮 | Slide 2 显示"展开答案"按钮；Slide 4 显示"展开三阶段"按钮（分三步，互斥展开） |

## 四、展示页规范

| # | 要求 | 说明 |
|:--:|------|------|
| **D1** | 字体放大 | 所有字体（h1、正文、卡片文字）放大一倍 |
| **D2** | 真实内容 | 所有页面用真实内容，禁止 placeholder |
| **D3** | 来源对照 | 内容对照演讲稿和本大纲逐页编写 |
| **D4** | 背景统一 | 默认黑底（全系列统一暗色背景） |

## 五、Slide 4 模型页交互（强制）

| # | 要求 | 说明 |
|:--:|------|------|
| **M1** | 互斥展开 | 三阶段卡片互斥展开——展开一张时其他自动收起 |
| **M2** | 分步控制 | 控制台分三步发送指令：card=1 → card=2 → card=3 |
| **M3** | 局限案例 | `mode="limitations"` 展示三个局限（幻觉/知识截止/无真实理解） |
| **M4** | 全部收起 | `expand=false` 全部收起 |

## 六、Slide 2 外星人语言交互（强制）

| # | 要求 | 说明 |
|:--:|------|------|
| **A1** | 默认隐藏 | 答案卡片默认隐藏，仅显示箭头序列 + 猜题区 |
| **A2** | 触发展开 | `expand=true` 展开答案卡片；`expand=false` 收起恢复猜题状态 |

## 七、API 交互

| # | 要求 | 说明 |
|:--:|------|------|
| **API1** | 控制台发送 | `{slide: N, expand: true/false, card: 1/2/3}` 到 API |
| **API2** | 展示页接收 | 展示页必须正确处理上述指令 |
| **API3** | 调试检查 | 控制台发指令后检查展示页是否收到 |

# 设计规范

> 📐 以下为设计层面的统一规范。

## 一、排版层级

| 层级 | 元素 | 说明 |
|:----:|------|------|
| **H2** | 页面主标题 | 页面核心主题，字号最大 |
| **H3** | 副标题 | 补充说明，字号次之 |
| **H4** | 内容标题 | 具体要点标题，字号再次之 |
| — | 内容正文 | 详细描述，正文字号 |

> 全篇保持 H2 > H3 > H4 的层级结构一致，不得出现层级倒挂。

## 二、四大工程揭晓页统一模板

| 区域 | 内容 |
|:-----|------|
| 左上 | CHAPTER 0X 标签（胶囊形）→ 大字标题（工程名称）→ 英文（ENGINEERING）→ 一句概述 |
| 左栏 | ❌ 没有该工程 → 问题描述 // ✅ 有该工程 → 效果描述 |
| 右栏 | 规则卡片 / STEP 卡片（对应工程的核心方法/步骤） |
| 底部 | 💡 核心提醒 + 🛠 生态栏 |

> ⚠️ 如果选用模板不支持左右分栏，可灵活调整排版，内容优先级 > 格式一致性。

## 三、生态栏分层递进

每个工程页的生态栏只显示到对应层级，越往下越基础，越往上越进阶：

```
循环工程页（完整堆叠）
  ⏰ Cron · 🤖 Agent · 📊 日志        ← 第4层：循环新增
  🐍 Python · 🔗 Coze · 🔧 Dify     ← 第3层：规训新增
  📄 MD · 📓 Obsidian · 📘 飞书       ← 第2层：上下文/知识库
  🧠 ChatGPT · Kimi · 豆包 · ···    ← 第1层：提示词
```

| 工程页 | 显示层级 |
|:------|:--------|
| 提示词工程页 | 仅第1层 |
| 上下文工程页 | 第1~2层 |
| 规训工程页 | 第1~3层 |
| 循环工程页 | 第1~4层（完整堆叠） |

# 技术规范

## 一、技术架构

```
控制台（control.html）                  展示页（display.html / ai_scroll.html）
  └─ POST /api/control/go                  └─ 轮询 GET /api/control/current
       {slide: N}                                → 渲染对应 Slide
  └─ POST /api/control/qa-show             └─ 轮询 GET /api/control/qa-overlay
  └─ POST /api/qa/ask                      └─ 轮询 GET /api/questions
  └─ GET /api/questions
```

## 二、语音问答功能

| 功能 | 说明 |
|:-----|------|
| 硬件 | 一台笔记本连接麦克风，预先打开监听页面 |
| 触发 | 演讲者在手机控制台点击按钮启动语音识别 |

## Slide 16 — 揭晓 · 循环工程

### 主标题
**循环工程**

### 副标题
CHAPTER 04 · LOOP ENGINEERING
做 → 查 → 修 → 再输出，直到满意

### 内容标题
#### 左栏：关键对比

**❌ 只有规训**
单次执行，AI 输出完就结束——有没有问题它不管

**✅ 加上循环**
做 → 查 → 修 → 再输出——AI 自己检查、自己修正，直到符合标准

#### 右栏：LEVEL 01~04

| 级别 | 名称 | 说明 |
|:----:|------|------|
| LEVEL 01 | 自查 | AI 输出后自己检查一遍 |
| LEVEL 02 | 复盘 | 多轮执行后总结规律 |
| LEVEL 03 | 会诊 | 多个 AI Agent 交叉检查 |
| LEVEL 04 | 绩效 | 量化评估 + 持续优化 |

#### 底部三栏展开（初始隐藏，控制台分三步展开）

**第一步：循环流程（左栏）**
输入 → 处理 → 输出 → 检查 → 发现问题？→ 修正 → 再输出 → 通过 → 交付

**第二步：现在痛点 · AI 没有眼睛（中栏）**
AI 看不到实际运行结果，不知道自己做对了还是做错了——这就是为什么需要循环工程，让它"长眼睛"

**第三步：未来方向 · 多 Agent 并行（右栏）**
多个 AI Agent 同时工作，互相检查、互相补位——一个写代码，一个做 Code Review，一个跑测试

#### 生态栏（第1~4层，完整堆叠）

ChatGPT · Kimi · Claude

- 模板：四部曲模板 + 底部三栏展开
- 三栏初始全部隐藏，控制台分三步触发展开
- 左上 CHAPTER 04 标签

| 日常场景 | 风险 | 解法 |
|:---------|:-----|:-----|

#### 💡 Pro 升级方案
**私有化部署** — 数据完全不离开你的网络。

### 设计要点
