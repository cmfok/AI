# 从会用AI，到用好AI · 早期PPT大纲

## 设计系统

> **风格**：瑞士国际主义 × 苹果极简
> - **底色交替**：大章节黑底，内容页白底
> - **主色调**：暖白 `#FAFAF8` / 近黑 `#0A0A0A`
> - **点缀色**：浅蓝 `#4A90D9` / 安全橙 `#FF6B35`（仅关键数据）
> - **字体**：Inter（英）+ Noto Sans SC / 微软雅黑（中），全程无衬线
> - **核心规则**：直角纯色、无圆角无阴影、图标放蓝色圆圈内
> - **标题**：封面居中，内容页左上，超大加粗

## 排版铁律（强制）

### 一、案例对比页（P07/P12/P13/P16）

每个落地案例页统一采用 **左右对比结构**，但呈现方式分两种：

**实时演示型（P07/P12）** — 嵌入两个并排对话框区域，现场复制Prompt、实时跑AI
**截屏/流程图型（P13/P16）** — 左❌右✅用提前准备的截图或流程图

```
实时演示页：                        截屏/流程图页：
┌──────────┬──────────┐          ┌──────────┬──────────┐
│ 对话框截图  │ 对话框截图  │          │ 人工/单次  │ 自动化/循环 │
│ （现场填充） │ （现场填充） │          │ （截图）   │ （截图）   │
└──────────┴──────────┘          └──────────┴──────────┘
  关键变量：____                    关键变量：____
```

- 左栏用橙色系表达警示，右栏用蓝色系表达达成
- 底部一行点明"关键变量"
- 翻页后接"揭晓工程原理"页
- 提示词/演示素材从演讲稿对应段落提取

| 页码 | 案例 | 呈现方式 | 备注 |
|:---:|------|:---:|------|
| P07 | 语音→PPT | 提前准备好 | 传统AI做的PPT vs 本场分享会PPT（元演示） |

采用 **三栏结构**：

```
┌──────────────┬──────────────────┬──────────────────┐
│ 循环流程       │ ⚠ 现在：痛点       │ 🔮 未来：方向       │
│ 当前怎么做     │ 卡在哪            │ 往哪走            │
│ （蓝）        │ （橙）            │ （蓝+橙过渡）      │
└──────────────┴──────────────────┴──────────────────┘
```

- 形成「现状 → 痛点 → 方向」认知闭环
- 右栏"未来设想"：多Agent并行 + 实时通讯对齐，对比当前单Agent串行瓶颈
