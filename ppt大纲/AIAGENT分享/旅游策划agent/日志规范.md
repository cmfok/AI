# 日志规范 — 旅行 Agent v5

> ⚠️⚠️⚠️ 铁规则：禁止编造假数据。所有数据必须来自真实 API/网页抓取，无法获取时标注"数据缺失"严禁填充。⚠️⚠️⚠️

> 本文档定义 Agent 运行时的日志级别、格式和输出位置。

---

## 一、日志级别

| 级别 | 符号 | 用途 | 何时使用 |
|------|------|------|----------|
| INFO | ℹ️ | 正常操作信息 | 工具调用、解析完成、方案保存 |
| WARN | ⚠️ | 可恢复异常 | 输入越界、降级触发、部分数据缺失 |
| ERROR | ❌ | 不可恢复错误 | FlyAI 全部失败、文件写入失败 |

---

## 二、日志输出位置

### 2.1 终端（stdout）

所有日志实时输出到终端，格式：

```
[{HH:MM:SS}] {级别图标} {级别} | {上下文}: {详细信息}
```

示例：
```
[14:30:01] ℹ️ INFO | 解析: 解析完成: 大理 5天 ¥20,000
[14:30:02] ℹ️ INFO | FlyAI: 🔧 调用 search-hotel --dest-name "大理"
[14:30:03] ⚠️ WARN | 酒店: 主策略失败，尝试关键词搜索
[14:30:05] ❌ ERROR | FlyAI: search-flight timeout after 25s
```

### 2.2 方案文件内嵌

执行日志嵌入方案 Markdown 底部的「Agent 执行日志」章节。

### 2.3 日志文件（可选）

如需持久化日志，可配置环境变量：

```bash
# 开启文件日志（暂未实现，供后续扩展）
export TRAVEL_AGENT_LOG=/var/log/travel_agent.log
```

## 三、日志内容规范

### 3.1 必须记录的时机

| 时机 | 级别 | 内容 |
|------|------|------|
| Agent 启动 | INFO | 版本号、用户查询 |
| 解析完成 | INFO | 目的地/天数/预算/偏好 |
| 每轮采集开始 | INFO | 当前循环数/上限 |
| 工具调用 | INFO | 工具名 + 参数（截断60字符） |
| 数据成功 | INFO | 数据项名 + 采集方式 |
| 数据降级 | WARN | 原策略失败原因 + 降级策略 |
| 数据失败 | ERROR | 数据项名 + 失败原因 |
| 方案保存 | INFO | 文件路径 + 耗时 |
| 校验告警 | WARN | 字段名 + 原始值 → 修正值 |

### 3.2 日志字段含义

| 字段 | 说明 | 示例 |
|------|------|------|
| 时间戳 | HH:MM:SS 格式 | `14:30:01` |
| 级别 | INFO/WARN/ERROR | `INFO` |
| 上下文 | 日志来源模块 | `解析`/`FlyAI`/`校验`/`保存` |
| 详细信息 | 自由文本 | `解析完成: 大理 5天` |


## 五、日志诊断指南

### 5.1 常见日志模式解读

#### 模式A：一切正常
```
[14:30:01] ℹ️ 解析完成
🎯 大理 5天 ¥20,000
  Loop 1: ✅🏨 ✅🚄 ✅🎯 ✅✈️ | 修复4 | 剩余缺口0
✅ 方案已保存
```
→ 4/4 数据全部 1 轮获取，正常case。

#### 模式B：部分降级
```
  Loop 1: ⏳🏨 ✅🚄 ✅🎯 ✅✈️ | 修复3 | 剩余缺口1
  [14:30:03] ⚠️ 酒店: 主策略失败，尝试关键词搜索
  Loop 2: ✅🏨 ✅🚄 ✅🎯 ✅✈️ | 修复1 | 剩余缺口0
```
→ 酒店第2轮降级成功，轻微警告，方案正常。

#### 模式C：数据缺失
```
  Loop 6: ✅🏨 ⏳🚄 ✅🎯 ⏳✈️ | 修复0 | 剩余缺口2
  [14:30:20] ❌ train: 所有策略均失败
  [14:30:22] ❌ flight: 所有策略均失败
```
→ 火车+机票始终无法获取，方案中标注 "⚠️ 建议飞猪APP查询"。
