# 输出标准 — 旅行 Agent v5

> ⚠️⚠️⚠️ 铁规则：禁止编造假数据。所有数据必须来自真实 API/网页抓取，无法获取时标注"数据缺失"严禁填充。⚠️⚠️⚠️

> 本文档定义 Agent 生成旅行方案的 Markdown 输出规范，确保每次方案质量一致。

---

## 一、输出文件命名

```
{目的地}{天数}天{N-1}夜_旅行方案.md
```

示例：`大理5天4夜_旅行方案.md`

保存位置：Agent 所在目录（`travel-agent/`）。

---

## 二、文档结构（7个必需章节）

### 2.1 必须包含的章节

| 序号 | 章节 | 说明 |
|------|------|------|
| 1 | 行程概览 | 目的地/天数/预算/出行偏好 表格 |
| 2 | 交通方案 | 机票/火车/自驾 三部分，每部分完整 |
| 3 | **住宿方案对比** | **核心章节**：3个方案，每晚详细优缺点 |
| 4 | 每日行程参考 | 按天列出景点+建议 |
| 5 | 预算概算 | 分项估算表格 + 预算状态 |
| 6 | 数据采集状态 | 4项数据成败状态表格 |
| 7 | Agent 执行日志 | 循环数/数据项/警告/错误/耗时 |

---

## 三、住宿方案输出规范（核心重点）

### 3.1 三方案强制要求

每次输出必须提供 **3 个酒店方案**，按价格档次分为：

| 档次 | 价格范围 | 标签 |
|------|---------|------|
| 经济实惠型 | ¥0~150/晚 | 💚 |
| 舒适品质型 | ¥151~350/晚 | 💙 |
| 亲子精选型 | ¥350+/晚 | 🧡 |

### 3.2 每个方案的必需内容

- **基本信息表**：价格（单晚 + 全程合计）、档次、位置、综合评分（⭐5星制）
- **✅ 优点**：至少 3 条，从价格/位置/设施/亲子适配维度分析
- **⚠️ 缺点**：至少 2 条，诚实标注风险点
- **综合评分**：基于价格+位置+亲子适配计算（0-100分）

### 3.3 优缺点描述规范

每条描述格式：`{emoji} {维度}：{具体说明}`

维度分类：
| Emoji | 维度 | 说明 |
|-------|------|------|
| 💰 | 价格 | 预算对比/性价比 |
| 📍 | 位置 | 交通便利度/近景点 |
| 🏷️ | 档次 | 酒店级别/设施 |
| 👶 | 亲子 | 儿童友好度 |
| 🛏️ | 房间 | 房型/双床确认 |
| ⭐ | 评分 | 用户口碑 |

### 3.4 综合推荐

章节末尾必须有推荐总结表：

| 如看重... | 推荐 |
|-----------|------|
| 💰 节省开支 | {最便宜酒店名}（¥{价格}/晚）|
| ⚖️ 均衡体验 | {最高分酒店名}（¥{价格}/晚）|
| 👶 亲子品质 | {最贵酒店名}（¥{价格}/晚）|

## 四、交通方案输出规范

### 4.1 机票（如有）

表格列：序号、航班号、价格

### 4.2 火车（如有）

表格列：序号、车次/线路、价格

### 4.3 🚗 自驾方案（语义区分：全程自驾 vs 当地租车）

> ⚠️ 人类说「自驾」有两个意思，Agent 必须区分并分别呈现：
> ① **全程自驾**：自己开车从家到目的地（如广州→大理 1600km）
> ② **当地租车**：飞机/高铁到目的地后，在当地租车自驾（如大理租车 ¥300~400/天）

#### 全程自驾（如适用）

表格列：距离、耗时、油费、过路费、合计

#### 当地租车（如适用）

表格列：日租金、租车天数、保险、当地油费、合计

#### 双方案对比表（当 drive_mode="both" 时）

| 维度 | 全程自驾 🚗 | 当地租车 🏎️ |
|------|-----------|----------|
| 交通方式 | {出发地}→{目的地} {距离}km | 飞机/高铁 + {目的地}租车 |
| 单程耗时 | {小时}h | 2.5h 航班 |
| 交通费用 | ¥{全程合计} | ¥{机票} + 租车 ¥{租车合计} |
| 亲子友好度 | ⭐⭐ 长途累 | ⭐⭐⭐⭐ 短途+随时停 |
| 适合场景 | 时间充裕 | **5天以内行程** |

## 八、质量 Criteria

### 8.1 完整性检查（自动验证）

- [ ] 7个必需章节全部存在
- [ ] 酒店方案 ≥ 3 个，每个含 ≥ 3 优点 + ≥ 2 缺点
- [ ] 每日行程 = 天数
- [ ] 预算状态明确

### 8.2 数据准确性

- [ ] 酒店价格已从 FlyAI 实时获取
- [ ] 交通数据有来源标注
- [ ] 缺失数据有明确提示（⚠️ 标记）

### 8.3 可读性

- [ ] 表格使用正确 Markdown 格式
- [ ] 专业术语首次出现附说明
- [ ] 缺点诚实标注，不隐瞒风险
