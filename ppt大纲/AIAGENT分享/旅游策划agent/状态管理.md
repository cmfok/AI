## 二、状态对象（AgentState）

### 2.1 字段说明

```python
class AgentState:
    def __init__(self):
        self.loop = 0           # 当前循环数
        self.errors = []        # 错误列表 [{ctx, detail, time}]
        self.warnings = []      # 警告列表 [{ctx, detail, time}]
        self.gaps_filled = []   # 成功采集的数据项
        self.gaps_remaining = [] # 剩余缺口
        self.start_time = 0     # 启动时间戳（用于计算耗时）
```

### 2.2 状态字段含义

| 字段 | 类型 | 说明 | 示例值 |
|------|------|------|--------|
| `loop` | int | 当前采集循环数 | 1~6 |
| `errors` | list[dict] | 致命错误记录 | `[{"ctx":"FlyAI","detail":"timeout"}]` |
| `warnings` | list[dict] | 警告记录 | `[{"ctx":"校验","detail":"天数超限"}]` |
| `gaps_filled` | list[str] | 已成功的数据项 | `["hotel(L1)","poi(L1)"]` |
| `gaps_remaining` | list[str] | 剩余缺口 | `["flight"]` |
| `start_time` | float | 启动时间戳 | `1720000000.0` |

## 三、采集阶段状态追踪

### 3.1 每轮采集后的状态快照

```
Loop {N}: ✅🏨 ✅🚄 ✅🎯 ⏳✈️ | 修复3 | 剩余缺口1
```

- ✅ = 已获取
- ⏳ = 缺口（未获取）
- 修复 = 本循环成功采集的项数
- 剩余缺口 = 仍未获取的项数

### 3.2 缺失数据标注

在 `gaps_filled` 中标注采集方式：

| 标注格式 | 含义 |
|----------|------|
| `hotel(L1)` | 第1轮主策略成功 |
| `hotel(降级)` | 降级策略成功 |
| （未录入） | 最终失败，未采集到 |

## 四、错误状态记录

### 4.1 错误分类

| 类别 | 上下文(ctx) | 是否致命 |
|------|------------|----------|
| FlyAI 超时 | `"FlyAI"` | 否（降级） |
| JSON 解析失败 | `"FlyAI"` | 否（降级） |
| 输入非法 | `"校验"` | 否（自动修正） |
| 文件写入失败 | `"保存"` | 是 |

### 4.2 错误记录格式

```python
{
    "ctx": "FlyAI",
    "detail": "search-hotel timeout after 25s",
    "time": "2026-07-03T14:30:05"
}
```

## 六、异常恢复策略

| 异常场景 | 恢复策略 |
|----------|----------|
| FlyAI 全部失败 | 输出空方案 + 错误摘要，标注全部 ⚠️ |
| 部分数据缺失 | 正常输出，缺失处标注 ⚠️ |
| 文件无法写入 | 打印方案到 stdout，标注 "保存失败" |
| Python 异常崩溃 | 打印 traceback + 当前状态快照 |
