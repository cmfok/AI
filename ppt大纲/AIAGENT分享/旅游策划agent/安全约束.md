# 安全约束 — 旅行 Agent v5

> 本文档定义 Agent 运行时的安全边界、输入校验规则和风险控制策略。

---

## 一、输入安全

### 1.1 参数注入防护

所有传入 FlyAI CLI 的用户输入必须经过以下清理：

```python
# ✅ 正确：参数用双引号包裹
f'search-hotel --dest-name "{dest}" --check-in-date {date}'

# ❌ 禁止：直接拼接未清理的输入
f'search-hotel --dest-name {raw_input}'
```

**清洗规则**：
- 目的地名仅保留中文字符和已知城市名
- 日期字段强制为正则 `\d{4}-\d{2}-\d{2}` 格式
- 不允许 `;`、`|`、`$()`、反引号等 shell 特殊字符

### 1.2 输入边界校验

| 参数 | 下限 | 上限 | 默认值 | 越界处理 |
|------|------|------|--------|----------|
| 天数 | 1 | 30 | 5 | 截断到边界 |
| 预算 | ¥1,000 | ¥500,000 | ¥10,000 | 截断到边界 |
| 日期 | 今天 | 今天+90天 | 今天+7天 | 自动回退 |

### 1.3 城市白名单

```python
ALLOWED_CITIES = [
    "昆明", "丽江", "大理", "西双版纳", "香格里拉", "腾冲", "普洱",
    "北京", "上海", "广州", "深圳", "杭州", "成都", "重庆", "西安",
    "三亚", "海口", "厦门", "青岛", "大连", "桂林", "张家界", "黄山",
    "南京", "武汉", "长沙", "苏州", "贵阳", "拉萨"
]
```

不在白名单中的城市 → 使用默认值「大理」并记录警告。

## 二、API 调用安全

### 2.1 频率限制

| 规则 | 值 |
|------|-----|
| 最大循环数 | 6 次 |
| 单次调用超时 | 25 秒 |
| 同类型查询最大重试 | 2 次（主策略 + 1 次降级）|

### 2.2 网络异常处理

| 异常类型 | 处理方式 |
|----------|----------|
| 超时 | 记录错误，降级到 keyword-search |
| 连接失败 | 记录错误，跳过该类查询 |
| JSON 解析失败 | 记录原始输出，降级 |
| 退出码非0 | 记录 stderr，降级 |

### 2.3 禁止行为

- ❌ 禁止修改服务器文件
- ❌ 禁止在 FlyAI 命令中拼接用户提供的文件路径
- ❌ 禁止输出 FlyAI 返回的完整敏感信息（如身份证号）
- ❌ 禁止使用 `rm`/`delete` 等危险命令

## 三、数据安全

### 3.1 隐私保护

FlyAI 返回数据中如包含以下字段，直接过滤不输出：
- 身份证号
- 手机号（保留前3后4）
- 完整地址（仅保留到街道）

### 3.2 输出安全

| 类型 | 策略 |
|------|------|
| 酒店名称 | 直接输出 |
| 酒店价格 | 直接输出（非敏感） |
| 航班号 | 直接输出 |
| 用户个人信息 | **禁止输出**（姓名/电话/身份证） |

### 3.3 文件写入安全

- 输出文件仅写入 Agent 所在目录（`travel-agent/`）
- 文件名仅包含中文+数字+下划线
- 不允许路径穿越（`../` 等）

```python
# ✅ 安全：固定输出目录
OUTPUT_DIR = os.path.dirname(os.path.abspath(__file__))
safe_name = re.sub(r'[^\w]', '', dest)  # 去除非安全字符
filepath = os.path.join(OUTPUT_DIR, f"{safe_name}{days}天_旅行方案.md")
```

## 四、风险评估矩阵

| 操作 | 危害等级 | 安全风险 | 数据敏感 | 综合 |
|------|---------|---------|---------|------|
| 调用 FlyAI | 低（只读查询） | 中（外网API） | 低（公开数据） | 可直接执行 |
| 保存 Markdown 文件 | 低（新增文件） | 低（本地操作） | 低（公开信息） | 可直接执行 |
| 用户输入解析 | 低（纯文本） | 低（本地） | 低（非敏感） | 可直接执行 |

所有操作均为「低风险」，无需额外确认。

## 五、合规声明

- 本 Agent 仅查询公开可用的旅游信息（飞猪平台）
- 不存储用户个人信息
- 不涉及支付/登录/认证操作
- 生成方案仅供参考，实际预订请以飞猪/携程平台为准
