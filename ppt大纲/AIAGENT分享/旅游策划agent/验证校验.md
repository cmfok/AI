## 一、校验流程

```
输入解析 → 输入校验 → 数据采集 → 数据校验 → 方案生成 → 输出校验
```

每个阶段都有独立的校验点，不合格则中止或降级。

### 3.2 酒店数据深度校验

```python
def validate_hotel_data(hotels: list) -> bool:
    """校验酒店数据是否可用"""
    if not hotels or len(hotels) == 0:
        return False
    # 至少1个有名称的酒店
    named = [h for h in hotels if h.get('name')]
    return len(named) >= 1
```

### 3.3 价格数据校验

```python
def validate_price(price_str: str) -> bool:
    """校验价格字符串可解析"""
    try:
        n = int(re.sub(r'[^0-9]', '', str(price_str)))
        return 10 <= n <= 99999  # ¥10~99999 合理范围
    except:
        return False
```

## 四、方案生成校验

### 4.1 结构性检查（生成后自动执行）

```python
def validate_output(md_content: str) -> dict:
    """校验输出 Markdown 的结构完整性"""
    checks = {
        "有行程概览": "## 一、行程概览" in md_content or "# 行程概览" in md_content,
        "有交通方案": "## 二、交通" in md_content or "交通方案" in md_content,
        "有住宿方案": "## 住宿方案" in md_content or "住宿" in md_content,
        "有每日行程": "## 每日行程" in md_content,
        "有预算概算": "## 预算" in md_content,
        "有执行日志": "## Agent 执行日志" in md_content,
        "酒店方案≥3": md_content.count("### 方案") >= 3,
        "有优缺点": "✅ 优点" in md_content and "⚠️ 缺点" in md_content,
    }
    return checks
```

### 4.2 质量阈值

| 指标 | 阈值 | 不达标处理 |
|------|------|-----------|
| 酒店方案数 | ≥ 3 | 标注 "⚠️ 酒店数据不足" |
| 优点数/方案 | ≥ 3 条 | 不足的标注原因 |
| 缺点数/方案 | ≥ 2 条 | 不足的标注原因 |
| 每日行程条目 | = 天数 | 不足的填"自由探索" |
| 章节完整性 | 7/7 章节 | 缺失的标注 "⚠️ 缺失" |

## 五、错误校验 vs 正常输出

### 5.1 数据完全缺失时

不生成方案，直接输出错误摘要：

```
# ❌ 方案生成失败

FlyAI 接口异常，无法获取旅行数据。

| 失败项 | 原因 |
|--------|------|
| 🏨 酒店 | timeout |
| 🎯 景点 | timeout |
| 🚄 火车 | timeout |
| ✈️ 机票 | timeout |

建议：检查网络连接，或直接使用飞猪APP搜索。
```

### 5.2 部分数据缺失时

正常生成方案，缺失项标注：

```
### ✈️ 机票

> ⚠️ FlyAI 未返回机票数据，建议飞猪APP查询。
```
