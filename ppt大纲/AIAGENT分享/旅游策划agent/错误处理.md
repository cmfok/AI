# 错误处理 — 旅行 Agent v5

> 本文档定义 Agent 运行时的错误分类、处理策略和降级链。

---

## 一、错误分类体系

### 1.1 三级错误

| 级别 | 符号 | 说明 | 示例 | 处理 |
|------|------|------|------|------|
| L1 警告 | ⚠️ | 可恢复，方案仍可生成 | 某数据项查询失败 | 标注缺失，继续 |
| L2 错误 | ❌ | 部分影响，结果降级 | FlyAI 全部超时 | 输出空方案+错误摘要 |
| L3 致命 | 💥 | Agent 无法继续 | 磁盘满/权限不足 | 终止，打印诊断 |

### 1.2 错误来源分类

| 来源 | 涵盖 |
|------|------|
| 输入层 | 用户查询解析失败、参数越界 |
| 工具层 | FlyAI 超时、网络不通、JSON 解析失败 |
| 生成层 | 数据不足以生成方案 |
| 输出层 | 文件写入失败、磁盘空间不足 |

## 二、各类错误处理详解

### 2.1 输入层错误

```python
# 场景1：纯乱码输入
q = "asdfghjkl"
# → 使用默认值（大理/5天/¥10000），记录 WARN

# 场景2：部分参数识别不到
q = "去旅游"
# → days/budget/dest 全部使用默认值，记录 WARN×3

# 场景3：越界值
q = "大理0天 预算0万"
# → days→1, budget→10000，记录 WARN×2
```

**代码位置**：`parse()` + `validate_input()`

### 2.2 工具层错误（FlyAI CLI）

```python
try:
    r = subprocess.run(f'{FLYAI} {cmd}', shell=True, 
                      capture_output=True, text=True, timeout=25)
except subprocess.TimeoutExpired:
    # L1: 记录 ERROR → 降级到 keyword-search
    state.add_error("FlyAI", f"超时 ({timeout}s)")
    return {"error": "timeout"}
except:
    # L1: 记录 ERROR → 降级
    state.add_error("FlyAI", str(e))
    return {"error": "fail"}
```

**降级链**：

```
search-hotel 失败
    ↓
keyword-search "{目的地}亲子酒店双床房推荐"
    ↓ 仍失败
标注 "⚠️ FlyAI无数据，建议飞猪APP查询"
```

### 2.3 JSON 解析错误

```python
except json.JSONDecodeError:
    state.add_error("FlyAI", f"JSON解析失败: {r.stdout[:200]}")
    # 保留原始输出到日志，便于排查
    return {"error": "json_parse"}
```

### 2.4 网络/连接错误

| 症状 | 诊断 | 处理 |
|------|------|------|
| Connection refused | FlyAI 服务未启动 | 记录 ERROR，跳过该类查询 |
| DNS 解析失败 | 网络断开 | 记录 ERROR，全部降级 |
| 超时 | 响应过慢 | 记录 ERROR，降级 |

### 2.5 生成层错误

```python
# 场景：所有数据项都失败
if all(g not in gaps for g in ["hotel","train","poi","flight"]):
    # 全部在 gaps 中 → 所有查询都失败
    return gen_error_report(gaps)  # 输出错误摘要而非方案

def gen_error_report(gaps):
    """数据全部失败时的错误报告"""
    return f"""# ❌ 方案生成失败

## 失败原因

FlyAI 接口异常，{len(gaps)}/4 数据项未能获取。

| 数据项 | 状态 |
|--------|------|
"""
    + "".join([f"| {g} | ❌ 失败 |\n" for g in gaps])
    + "\n建议：检查网络连接，或直接使用飞猪APP搜索。\n"
```

### 2.6 输出层错误

```python
try:
    with open(filepath, 'w', encoding='utf-8') as f:
        f.write(plan)
except PermissionError:
    print(f"❌ 无法写入文件: {filepath}（权限不足）")
    print("方案内容如下：\n")
    print(plan)  # 降级为 stdout 输出
except OSError as e:
    print(f"❌ 文件写入失败: {e}")
    print(plan)
```

## 三、错误日志格式

### 3.1 终端输出

```
[14:30:02] ⚠️ WARN | 酒店: 主策略失败，尝试关键词搜索
[14:30:05] ❌ ERROR | FlyAI: search-flight timeout after 25s
[14:30:06] ❌ ERROR | 酒店: 所有策略均失败
```

### 3.2 方案文件内嵌

错误详情写入方案底部「Agent 执行日志」章节：

```markdown
### 错误详情

- ❌ **FlyAI**: search-flight timeout after 25s
- ❌ **酒店**: 所有策略均失败

### 警告详情

- ⚠️ **校验**: 天数 0 无效，重置为5
- ⚠️ **FlyAI**: search-train 退出码 1: network error
```

## 四、降级决策树

```
查询{数据项}
    │
    ├─ 成功 → 记录到 results，移除 gaps
    │
    └─ 失败 → 检查失败原因
            │
            ├─ 超时 → 降级 keyword-search
            │       ├─ 成功 → 记录，标注"(降级)"
            │       └─ 失败 → 标注 "⚠️ 建议XXAPP查询"
            │
            ├─ JSON错误 → 降级 keyword-search（同上）
            │
            └─ 其他 → 降级 keyword-search（同上）
```

---

## 五、错误恢复后的状态一致性

Agent 遵循「尽最大努力输出」原则：

1. **部分数据成功** → 输出完整方案，缺失处标注 ⚠️
2. **全部数据失败** → 输出错误摘要，不伪造数据
3. **工具全部不可用** → 输出空方案 + 诊断建议
4. **文件写入失败** → 降级为 stdout 输出

**核心承诺**：无论什么错误，Agent 都不会静默退出——要么输出方案，要么输出错误报告。
