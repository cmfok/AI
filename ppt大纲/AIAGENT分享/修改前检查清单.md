# AI 分享会 · 修改前检查清单

> **定位**：修改 AI 分享会项目代码前的必读检查清单。
> **范围**：`分享会/AI/` 下所有文件、服务器 `/home/ubuntu/share-ppt/ai-agent/` 的部署文件。
> **规则**：**修改任何代码前必须逐条过一遍本清单**，确认不违反后再操作。

---

## 一、部署确认

| # | 检查项 | 说明 | 示例 |
|---|--------|------|------|
| 1 | □ 确认服务器 SSH 地址 | 用 `my-server` 别名（指向 `106.53.168.220`），禁用 Cloudflare 域名直连 | ❌ `ssh ubuntu@8008.cmfok.online` → Cloudflare IP，SSH 不通
✅ `ssh my-server` → HostName=106.53.168.220, User=ubuntu, Port=22 |
| 2 | □ 确认本地文件是真相源 | 本地文件直接修改，不要从服务器下载到 tmp 再改 | 正确的流程：改 `分享会/AI/ppt大纲/AIAGENT分享/control.html` → 直接上传覆盖 |
| 3 | □ 备份原始文件 | 修改服务器文件前先备份到 `.bak` | `sudo cp file.html file.html.bak` |
| 4 | □ 确认部署路径与 Flask 路由匹配 | 查 `app.py` 中 `static_folder` 指向的目录，文件要部署到对应位置 | `app.send_static_file('control.html')` → 放 `PPT_DIR/` 根目录，不是 `ai-agent/` |
| 5 | □ 确认服务器部署目录 | PPT_DIR=`/home/ubuntu/share-ppt/`，所有静态文件和 `app.py` 在此目录 | SCP 目标路径：`ubuntu@106.53.168.220:/home/ubuntu/share-ppt/` |
| 6 | □ 确认域名根路径部署 | 项目域名 `https://8008.cmfok.online/`，文件直接部署到根路径，禁止 `/ai/` 子路径 | ✅ `https://8008.cmfok.online/control.html`
❌ `https://8008.cmfok.online/ai/control.html`
❌ 使用 `8008.cmfok.online` 域名 SSH 连接 |

## 二、备份铁律（不可违反）

| # | 检查项 | 说明 |
|---|--------|------|
| B1 | □ **任何修改前必须备份原始文件** | 无论修改范围大小（哪怕改 1 行），修改前必须备份到 `.bak`，禁止以"改动小"为由跳过备份 |
| B2 | □ **脚本写入前必须验证** | 使用脚本写入文件前，先备份、再在副本上测试通过后，才能写入正式文件 |
| B3 | □ **文件损坏第一时间恢复** | 修改导致文件损坏、变空、内容异常时，立即从服务器 `.bak` 或 git 恢复，不得擅自继续操作 |

## 三、代码修改

| # | 检查项 | 说明 |
|---|--------|------|
| 7 | □ 无语法错误 | 检查拼写错误（如 `falfunction` → `false`）、括号配对、引号配对 |
| 8 | □ Python 布尔值大小写 | Python 用 `True`/`False`，不是 `true`/`false` |
| 9 | □ 函数引用存在性 | 所有调用的函数、变量在哪个 `<script>` 块/作用域中定义，确保可访问 |
| 10 | □ 未定义调用保护 | 跨脚本块调用的函数加 `typeof fn === 'function'` 保护 |
| 11 | □ 异步时序无竞态 | 多个异步操作先后执行时，不会因顺序不确定导致状态不一致 |
| 12 | □ 变量声明后调 UI 更新函数 | 声明状态变量后立即调用对应的 UI 渲染函数，确保初始状态有样式 | `var locked=false;` 后要调 `updateLockBtn()` |

## 七、演讲稿（铁规则）

| # | 检查项 | 说明 |
|---|--------|------|
| 21 | □ **修改任何文件后必须验证演讲稿加载** | 在浏览器打开控制台 → 翻页 → 查看演讲稿面板是否能正确显示对应幻灯片的内容 | ❌ 演讲稿挂了 = 没有检查 |

| 23 | □ `cache` 模式改为 `no-store` | `{cache:'no-store'}` 直接绕过缓存，避免 304 响应 | ✅ `fetch('speech.md',{cache:'no-store'})` |
