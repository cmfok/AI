### 2. 数据来源与加载

#### 2.1 四档价格预加载（对账前必须执行）

【重要】每次对账开始时，必须先读取报价单、展示当月档位价格给CM确认，确认后四档价格锁定到内存，后续计算直接引用，不再访问报价单文件。

【强制】禁止在预加载流程中硬编码价格
- 所有价格必须从当月报价单中实时查询
- 禁止使用规则手册中的示例价格
- 禁止使用上一个月确认的价格

预加载流程：
1. 读取报价单（外仓报价*.xlsx）
2. 计算当月档位（日均票数 = (广州仓+无锡仓)/当月天数）
3. 展示当月档位价格给CM确认
4. CM确认后，四档价格写入内存变量
5. 执行对账 → 计算时直接用内存中的价格
6. 对账结束 → 价格变量释放

【重要】禁止硬编码价格
- 规则中只写计算公式和档位阈值，禁止写入具体价格数值
- 每月的档位价格必须从报价单中查找当月实际档位对应的价格

### 6. 订单与运单关系规则

#### 规则6.1：一个运单号 = 一个包裹 = 一个包装方案

【核心】一个运单号对应唯一一个包裹，使用唯一一个包装方案。

规则：
- 一个运单号 → 一个发货批次 → 一个包装方案
- 多类产品混合 → 也只能使用一个包装方案（不能拆分多个包装方案）

禁止行为：
- ❌ 一个运单拆分多个包装方案
- ❌ 多个运单合并一个包装方案

#### 规则6.2：顺丰账单订单号 ≠ 内部订单号（强制）

【核心】顺丰账单上的订单号字段是顺丰自己生成的，与我们的内部订单号不同。

正确关联方式：
- 顺丰账单.订单号 → 通过 fact_shipping_detail_2026.原始单号 关联
- 不能直接用顺丰账单.订单号 查询 fact_order_detail

#### 规则6.3：运单拆分核查（强制）

【核心】1个运单号=1个包裹=1个包装方案，但需拆分时必须核查订单运单数。

判断流程：
- 运单号需要拆分包装方案时
- 查找该运单号对应的订单号
- 在订单明细表查该订单号的所有运单记录
  - 运单记录总数大于1 → 属于正常拆分，可以执行
  - 运单记录总数等于1 → 异常，停止并上报CM

数据来源：
- fact_order_detail.订单号 → 关联多个运单号
- fact_shipping_detail_2026.原始单号 → 关联多个运单号

禁止行为：
- 拆分时未核查订单运单数
- 拆分时仅看顺丰账单（顺丰账单不显示订单内多个运单）

### 7. 执行结果验证机制

### 9. 包装方案选择与处理流程（整合版）

#### 规则9：包装方案完整处理流程

【核心】包装方案处理分为两个阶段：第一阶段（五步走）正常匹配，第二阶段（匹配失败时）处理多品类/单品类/特殊品类。

---

##### 第一阶段：五步走（步骤1-5）

**第一步：统计每个分类的包数**
- 根据订单中每个SKU的产品分类，统计该分类的总包数
- 数据来源：dim_product_category（货品编号→产品分类）
- 输出：{A类:n包, E类:m包, I类:p包, ...}

**第二步：生成包装方案编码**
- 格式：[数量A][A][数量E][E][数量I][I][数量J][J][数量K][K][数量P][P][数量Q][Q][数量Y][Y][数量Z][Z]
- 按固定顺序排列（AEIJKPQYZ），没有的分类用0占位
- 示例：1个A类 + 2个E类 → "1A2E0I0J0K0P0Q0Y0Z"

**第三步：在包装耗材映射表中查找**
- 数据来源：数据库 dim_packaging_plan 表
- 通过包装方案编码匹配"配置逻辑"字段
- **验证结果决定后续流程**：
  - 匹配成功 + 配置逻辑≠"禁用" → 进入步骤4/5（正常五步走）
  - 匹配失败（禁用/找不到精确匹配） → **进入第二阶段**

**第四步：确定实际时效**
- 数据来源：数据库 city_coverage时效 表（key=省份|城市）
- 根据发货仓库选择"广州仓时效"或"无锡仓时效"
- 验证：时效为空 → 停止上报

**第五步：替换时效，得到最终方案**
- 用实际时效替换包装方案编码中的时效
- 验证：最终方案存在且配置逻辑≠"禁用"

---

##### 第二阶段：匹配失败时的处理（步骤A/B/C）

**触发条件**：步骤3匹配失败（配置逻辑="禁用"或找不到精确匹配）


**步骤A：判断品类数量**
- 统计订单中涉及的分类数量
- 单品类（只有1个分类）→ 直接进入【单品类处理】
- 多品类（2个及以上分类）→ 进入【多品类处理】

**步骤B：特殊品类处理（多品类时执行）**
- 判断是否存在Y/Z类特殊品类
  - **存在Y/Z类**：
    - 计算Y/Z类的干冰数量（干冰500g配置值）
    - 作为整个包裹的**最终干冰数量**
    - 其他品类只使用冷冻冰250g
  - **不存在Y/Z类**：
    - 干冰数量 = 0
    - 所有品类只使用冷冻冰250g

**步骤C：多品类合并为单包裹编码（多品类时执行）**
1. 统计**体积关系最大的分类**（决定用多大的箱/袋）
2. 统计该订单的**总包数**（所有品类包数相加）
3. 组合成单品类包装编码（如"3K"：最大分类K，总包数3）
4. 用新编码重新走回【单品类处理流程】

**分类体积优先级（从小到大）**：

| 优先级 | 分类 | 说明 |
|--------|------|------|
| 1（最小） | A、Z | A与Z体积相同，Z为特殊品类 |
| 2 | E、Y | E与Y体积相同，Y为特殊品类 |
| 3 | I | — |
| 4 | J | — |
| 5 | K | — |
| 6 | P | — |
| 7（最大） | Q | 体积最大 |

- 体积大小按字母顺序排列：A最小，Q最大
- Y/Z为特殊品类，体积分别与E/A相同，但拥有独立的干冰计算步骤
- 合并时取体积最大的分类作为主导分类，决定箱型/保温袋型号

##### 单品类处理（单包裹/多包裹判断）

```
方案上限 = 包装耗材映射表[方案]["单包裹数量上限"]
产品总数量

判断：
  产品总数量 ≤ 上限 → 单包裹 → 重量计算 → 运费计算
  产品总数量 > 上限 → 多包裹拆分 → 各包裹独立计算 → 汇总
```

##### 包装耗材重量计算

**实际重量公式**：
```
产品净重 = 规格名称正则提取(Xg) × 数量
包装耗材重量 = 保温袋重量 + 泡沫箱重量 + 干冰重量 + 冷冻冰重量
实际重量 = 产品净重 + 包装耗材重量
计费重量 = ceil(实际重量 × 10) / 10
```

**耗材重量来源**：
- 从数据库 `dim_material_cost`（耗材成本表）读取
  - 匹配字段：`short_name`（规格简称，如 S1号/S1）+ `material_name`（材料名，如 泡沫箱/保温袋/冰袋/干冰）
  - 重量字段：`weight_g`（克，除以1000转为kg）
- 所有耗材重量均从表中读取，禁止在代码或规则中硬编码

**Y/Z类干冰配置**：
- Y类、Z类产品使用干冰500g + 冷冻冰250g
- JSON配置字段：干冰500g（例：1-5个）、冷冻冰250g（例：2-7个）
- 其他分类（A/E/I/J/K/P/Q）：只使用冷冻冰250g，干冰=0

### 11. 阶梯定价规则

#### 规则11：阶梯定价规则

【核心】根据日均票数确定档位，同一档位内按仓库+分区查不同价格。

【重要】禁止硬编码价格
- 规则中只写计算公式和档位阈值，禁止写入具体价格数值
- 每月的档位价格必须从报价单中查找当月实际档位对应的价格
- 严禁在规则中预设"四档价格=4.12元"等硬编码

统计范围：
- 仅统计广州仓 + 无锡仓的包裹数
- 佛山工厂发货（电商仓）不纳入统计

计算公式：
日均票数 = (广州仓月包裹总数 + 无锡仓月包裹总数) ÷ 当月天数

档位划分：

| 档位 | 日均票数 | 说明 |
|------|----------|------|
| 一档 | 30(含)-200票/天 | 日均30-200票 |
| 二档 | 200(含)-500票/天 | 日均200-500票 |
| 三档 | 500(含)-1000票/天 | 日均500-1000票 |
| 四档 | 1000(含)-2000票/天 | 日均1000-2000票 |
| 五档 | 2000票/天 | 日均2000票以上 |

【重要】同一档位内价格查询逻辑：
- 档位确定后，按 仓库 + 目的地省份所属分区 查找具体价格
- 无论发货地区、收货地区、时效、包装方案如何，都不影响档位内的分区价格
- 价格必须从当月报价单中查找当前档位对应的价格，禁止使用上一个月的价格

价格来源： 外仓报价*.xlsx（当月报价单）

档位计算示例：
- 某月广州仓包裹数：X票
- 某月无锡仓包裹数：Y票
- 合计：X+Y票
- 当月天数：Z天
- 日均票数 = (X+Y) ÷ Z
- 根据日均票数确定档位

【强制】档位价格查询流程（每月必须执行）：
1. 根据当月总运单数确定档位（计算日均票数）
2. 从当月报价单中查找该档位对应的各仓库各分区价格
3. 将价格展示给CM确认，确认后锁定到内存再执行对账
4. 严禁直接使用上一个月的价格或规则中预设的价格

### 12. 分段计费公式

#### 规则12：分段计费公式

【核心】运费按实际包裹重量分段计算。

计算公式：
运费 = 首重价格 + 续重1×分量1 + 续重2×分量2 + 续重3×分量3

分段规则：

| 分段 | 重量范围 | 单价 |
|------|----------|------|
| 首重 | 前1kg | 首重价格 |
| 续重1 | 1-3kg（2kg） | 续重1 × 2kg |
| 续重2 | 3-5kg（2kg） | 续重2 × 2kg |
| 续重3 | 5kg以上 | 续重3 × 超出部分 |

示例1（2kg包裹，广东一区四档）：
- 实际包裹重量：2kg
- 分段：
  - 第1kg：首重4.12元
  - 第2kg：续重1×1 = 1元
- 应收运费：4.12 + 1 = 5.12元

示例2（6.8kg包裹，广东一区四档）：
- 实际包裹重量：6.8kg
- 分段：
  - 第1kg：首重4.12元
  - 第2-3kg（2kg）：续重2×1 = 2元
  - 第4-5kg（2kg）：续重2×1.5 = 3元
  - 第6-7kg（1.8kg）：续重3×2 = 3.6元
- 应收运费：4.12 + 2 + 3 + 3.6 = 12.72元

### 13. 实际包裹重量计算

#### 规则13：实际包裹重量计算

【核心】实际包裹重量 = 产品净重 × 数量 + 包装耗材重量

计算公式：
实际包裹重量 = 产品净重 × 数量 + 包装耗材重量

包装耗材构成：

| 耗材类型 | 说明 |
|----------|------|

| 泡沫箱 | 固定1个 |
| 冰袋 | 按配置（冷冻冰250g/500g） |
| 干冰 | 按配置 |

计费重量计算：
计费重量 = ceil(实际包裹重量 × 10) / 10
（向上取整到0.1kg）

- ❌ 直接使用顺丰账单上的实际包裹重量

#### 【体积重计算说明（2026-05-25更新）】

- 我方计算(体积重)(元)：取MAX(实际重量, 体积重量)作为计费重量重新计算运费
- 我方应付(元) = MAX(我方计算(元), 我方计算(体积重)(元))：取实际重运费和体积重运费中的较大值作为最终应付金额
- 差异(元) = 我方应付(元) - 顺丰应付(元)：以较大值为基准计算差异
- 两种价格同时输出「我方计算(元)」和「我方计算(体积重)(元)」，最终用「我方应付(元)」进行对账
- 体积重数据来源于dim_material_cost表的material_spec字段（如"29cm*17cm*19cm"），由顺丰提供
- 轻抛系数：6000（顺丰标准）
